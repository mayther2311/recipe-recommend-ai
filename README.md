# 🍳 献立提案アプリ (Menu Suggestion AI)

冷蔵庫の画像から食材を認識し、AI（Gemini 3 Flash）が最適な献立を提案するWebアプリケーションです。

## 🚀 主な機能
- **画像解析**: 冷蔵庫の画像から、肉・魚・野菜などの主要食材を自動特定。
- **複数献立の同時提案**: 1枚の写真から、3つの異なるレシピを提案。
- **詳細な栄養情報**: 1人前の推定カロリーおよび、摂取できる主要栄養素を表示。
- **買い物リスト**: 一般的な調味料を除外し、料理に必要なメイン食材のみを「不足リスト」として抽出。
- **リソース最適化**: プライバシーとストレージ保護のため、解析完了後にサーバーから画像を自動削除。

## 🛠 技術スタック
### Frontend
- **Framework**: Flutter (Material 3)
- **Language**: Dart

### Backend
- **Framework**: FastAPI (Python 3.12)
- **Language**: Python
- **AI API**: Google Gemini 3 Flash Preview


🏗 システムの仕組みとデータフロー
本アプリケーションは、フロントエンド、バックエンド、AI APIの3層構造で構築されており、以下のステップで「写真から献立を作る」という体験を実現しています。

1. フロントエンド：ユーザーインターフェース (Flutter / Dart)
ユーザーがアプリ上で冷蔵庫の写真を撮影、またはギャラリーから選択するところから始まります。

使用技術: Flutter (Material 3)

役割: 画像の取得と、バックエンドサーバーへの安全なデータ転送を担います。

2. バックエンド：リクエスト受付と一時保存 (FastAPI / Python)
サーバーがフロントエンドから画像を受け取ります。

使用技術: FastAPI (Python)

役割: 送られてきた画像をサーバー内の static フォルダへ一時的に書き込みます。この一時保存により、AIが画像を読み取れる状態を整えます。

3. AIによる画像解析 (Google Gemini 3 Flash API)
保存された画像をAIへ送り、内容を解析します。

使用技術: Gemini 3 Flash


4. 構造化データの返却とリソース管理
AIから返ってきた解析結果を、サーバーが受け取ります。

データ形式: 料理名、手順、カロリー、栄養素が整理された状態。

自動クリーンアップ (finallyブロック):
解析が完了した瞬間に、サーバー内に一時保存していた画像ファイルを即座に削除します。これにより、ユーザーのプライバシー保護と、サーバーのストレージ容量の節約を両立させています。

5. フロントエンド：動的UI表示
最後に、サーバーから届いたデータをユーザーの画面に表示します。

役割: データを解析し、複数の「献立カード」として表示します。
